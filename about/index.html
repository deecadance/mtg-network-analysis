About the Method

Archetypes first. Then win-rate deltas, uncertainty, and draft pressure.

The site does not start from a global pair table. It learns archetypes from full decklists, then measures single-card strength, pair synergy, triplet motifs, and draft urgency inside each learned shell.

This page is intentionally mathematical. The formulas below are the ones the current implementation is built around.

1. Archetypes from full decklists

Let X be the deck-by-card matrix built from the deck_* columns. Cards that appear in more than 55% of decks are filtered before clustering so they do not swamp the geometry.

w(d,c) = (1 + log x(d,c)) * log((1 + D) / (1 + df(c)))

The weighted deck matrix is reduced with truncated SVD and clustered with MiniBatch K-means. The automatic search currently scans k = 7..11 and keeps the best fit subject to a minimum large-cluster size.

2. Signposts

For archetype k, a card is a signpost when it appears much more often inside that archetype than outside it. The method uses smoothed log-odds with a prevalence-centered prior and mild rarity damping.

ell(c) = log((a_c + alpha_c) / (N_k - a_c + beta_c)) - log((b_c + alpha_c) / (N_not_k - b_c + beta_c))
z(c) = ell(c) / sqrt(Var(ell(c)))
signpost_score(c) = z(c) * sqrt(a_c)

Here a_c is the number of decks in the archetype containing the card and b_c is the number of decks outside the archetype containing it.

3. Deck-level motif lift

Deck-level pair and triplet lift ask whether cards co-occur in completed decks more often than independence would predict.

E_deck(a,b) = N_k * pi(a) * pi(b)
deck_pair_log_lift(a,b) = log2((O_deck(a,b) + 1) / (E_deck(a,b) + 1))
deck_pair_quality(a,b) = deck_pair_log_lift(a,b) * sqrt(O_deck(a,b))
E_deck(a,b,c) = N_k * pi(a) * pi(b) * pi(c)
deck_triplet_log_lift(a,b,c) = log2((O_deck(a,b,c) + 1) / (E_deck(a,b,c) + 1))

Strong deck triplets feed a pair bonus back into the graph, so higher-order motifs can reinforce a pair without redefining the game-level synergy statistic itself.

4. Game-level pair synergy

Inside each archetype, the baseline is the archetype win rate p_k. For a card a, let p_a be its in-zone win rate. For a pair (a,b), let p_ab be the win rate when both are present in the chosen zone.

standalone_delta(a) = p_a - p_k
logit(p_hat_ab) = logit(p_a) + logit(p_b) - logit(p_k)
synergy_delta_logit(a,b) = logit(p_ab) - logit(p_hat_ab)

This is the core edge statistic. Positive values mean the pair wins more than its two single-card rates would predict inside the same archetype.

5. Standard deviation and confidence

Sampling noise is estimated with binomial variance in probability space and with the corresponding delta-method approximation in logit space.

Var(p) = p * (1 - p) / n
Var(logit(p)) = 1 / (n * p * (1 - p))
SE_delta_logit(a,b) = sqrt(Var(logit(p_ab)) + Var(logit(p_hat_ab)))
sigma(a,b) = abs(synergy_delta_logit(a,b)) / SE_delta_logit(a,b)
confidence_probability(a,b) = erf(sigma(a,b) / sqrt(2))

The site shows confidence as a percentage. That percentage is just the error-function mapping of the sigma statistic above.

6. Pair, triplet, and draft corroboration scores

The graph does not rank pairs by one number alone. It multiplies game lift, support, deck-level corroboration, triplet structure, draft corroboration, and confidence.

pair_quality_raw(a,b) = max(0, synergy_delta_logit(a,b)) * sqrt(n_ab) * (0.02 + max(0, p_ab - p_k)) * (1 + 0.25 * max(0, deck_pair_log_lift(a,b))) * (1 + 0.18 * max(0, triplet_bonus(a,b))) * (1 + 0.05 * log(1 + draft_pair_events(a,b)))
pair_quality(a,b) = pair_quality_raw(a,b) * min(1, sigma(a,b))
pair_pick_score_raw(a,b) = sqrt(n_ab) * (0.02 + max(0, p_ab - p_k)) * (1 + 0.55 * max(0, deck_pair_log_lift(a,b))) * (1 + 0.10 * max(0, triplet_bonus(a,b))) * (1 + 0.06 * log(1 + draft_pair_events(a,b)))
triplet_quality_raw(a,b,c) = synergy_delta_logit(a,b,c) * sqrt(n_abc) * (1 + 0.20 * max(0, deck_triplet_log_lift(a,b,c))) * (1 + 0.06 * log(1 + draft_triplet_events(a,b,c)))

7. Draft pressure, ALSA, and wheeling

ALSA is global, not archetype-specific. It measures what the whole table is doing, not just what one lane wants.

ALSA(c) = (1 / N_c) * sum_i last_seen_i(c)
ALSA_sd(c) = max(0.65, sqrt((1 / N_c) * sum_i last_seen_i(c)^2 - ALSA(c)^2))
tau(c) = max(1.1, 0.60 * ALSA(c) + 0.80 * ALSA_sd(c))
wheel_model(c) = (1 / 4) * sum_(s=1..4) exp(-max(0, (s + 8) - ALSA(c)) / tau(c))
wheel_probability(c) = max(0.12 * wheel_model(c), (empirical_wheels(c) + 320 * wheel_model(c)) / (wheel_opportunities(c) + 320))

The site only reports the probability that the card comes back when first seen on picks 1 through 4.

8. All node scores

Dependency score Weighted mean of positive incident pair synergy and triplet bonus. High means the card mostly looks good when its neighbors are present.
Analysis score
analysis_raw(c) = signpost_score(c) + 18 * dependency(c) + 140 * max(0, standalone_delta(c))
Then transformed with sign(x) * log(1 + |x|), clipped at the 5th and 95th percentiles, and rescaled to 0-10 inside the archetype.
Pick priority score
pick_priority_raw(c) = 100 * (max(0, draft_win_lift(c)) + 0.55 * max(0, standalone_delta(c))) * sqrt(max(1, draft_drafts(c))) * (1 + 0.10 * max(0, signpost_lift(c) - 1))
Then transformed with log(1 + x), clipped, and rescaled to 0-10.
Pick market score Built from ALSA, average taken-at, and wheel probability. This is global table pressure, not archetype power.
Pick urgency
pick_urgency(c) = 10 * clamp((0.90 * market(c) + 0.10 * pick_priority(c) - 0.58) / 0.42, 0, 1)^1.15
Plus a saturation rule for true slam-first-pick cards.
Shell dependence
shell_dependence_raw(c) = max(0, support_delta(c)) * sqrt(max(1, supported_games(c))) * (1 + 4 * dependency(c)) * (1 + 0.18 * max(0, signpost_lift(c) - 1)) / (1 + 16 * max(0, standalone_delta(c)))
Then normalized to 0-10 with the nonnegative log-rescale.
Build-around flag
supported_games >= 80 support_delta >= 0.018 dependency >= 0.075 standalone_delta <= 0.014
A high shell-dependence score alone is not enough.
Core score
core_raw(c) = signpost_score(c) + 52 * deck_prevalence(c) + 85 * max(0, standalone_delta(c)) + 14 * dependency(c)
Then normalized to 0-10.

Displayed solo/supported statsNeed at least 30 games in the relevant bucket or the UI shows “Not enough data”.
Base edge thresholdsynergy_delta_logit >= 0.05 and seed confidence around 90%.
Knot thresholdWeighted internal-edge confidence around 80%.
Knot promotionOnce a knot exists, nearby edges can enter around 81% confidence if they reinforce that knot and do not bridge two separate certified knots.
UI edge filterThe current front-end asks for about 85% confidence or explicit knot promotion.
Arrow directionHigher-win-rate card points to lower-win-rate card. The pair statistic itself remains symmetric.

10. What the graph means

  • Big node: high score in the active view.
  • Bright thick arrow: strong positive pair evidence with good confidence.
  • Knot: a community of positive edges whose internal support is strong enough to survive the knot-confidence filter.
  • Isolated node: either the card is good on rate by itself or the pair evidence around it did not survive the current thresholds.

11. Limits

The method is descriptive, not causal. Archetypes are hard clusters. Solo and supported win rates are conditional slices, not interventions. Confidence tracks sampling uncertainty under the modeled estimators, not every possible source of bias in 17Lands data. The point of the site is still practical: learn what the decks are, what holds them together, and which cards improve when they are paired with the right support.
